## Contexto e Objetivo
Finalização do Débito 3 da feature 001-gate-migrations-refactor. O objetivo era configurar testes de integração (`integration_apply.sh`) rodando contra um container descartável do Postgres remotamente na VM via SSH.

## O Que Foi Feito
- **Commit 1:** Suporte dinâmico para `MIGRATIONS_DIR` implantado. Unit testes (bats) GREEN (12/12).
- **Commit 2:** Fixtures de testes e o compose yaml foram mockados na working tree, e validamos o Compose via dry-run no servidor.
- **Commit 3 (Ação atual):** 
  - O script `integration_apply.sh` envia um tarball, sobe o banco de dados remoto e alimenta as asserções de teste.
  - Para permitir testes paralelos isolados, injetamos a variável de ambiente opcional `COMPOSE_PROJECT` usando a flag explícita `-p` nos chamados do Docker Compose (`Correção A` confirmada por testes de diagnóstico separados).

## Falha Identificada (Bug do Stdin)
Ao rodar os testes remotos:
- O teste executou `docker compose exec -T ... psql` (dentro de `apply_required_migrations.sh`).
- O docker compose devorou (consumiu) a `stdin` que estava provendo o script de bash via heredoc (`ssh bash << REMOTE_SCRIPT`).
- O shell remoto, desabastecido de instruções, assumiu que o script havia finalizado com `exit 0` prematuramente. 
- A rotina de `[cleanup]` foi disparada logo no "TEST 1", passando no healthcheck mas cortando os demais passos.

## Próximos Passos
1. Adicionar os redirects `< /dev/null` a **todas** as queries `psql` que dependem de `docker compose exec` (principalmente no `lib_migrations.sh` e `apply_required_migrations.sh`) para prevenir que engulam a entrada do Heredoc.
2. Re-rodar o `bash testes/deploy/integration_apply.sh`.
3. Certificar os 6 testes de integração GREEN.
4. Efetuar o **Commit 3** real do teste de integração com sucesso.
