## Débitos/riscos desta spec (ficam aqui, dentro da spec 084 — não duplicar em `specs/backlog.md`)
- [ ] D-084-01 — Lacuna jurídica: implicação de OGL/ORC sobre redistribuição de arquivo completo (vs. só texto de regras) não resolvida por pesquisa; precisa de análise jurídica dedicada antes de considerar resolvida.
- [ ] D-084-02 — Risco de suspensão de conta pessoal do mantenedor ao usar Modo 3 (script local com sessão logada real) para scraping — aceito conscientemente por decisão do mantenedor, mas deve ficar visível/rastreável, não escondido.
- [ ] D-084-03 — Detecção automática de idioma cobre só título/descrição submetidos, nunca o conteúdo do arquivo linkado — moderação humana continua sendo a única defesa contra esse caso específico; registrar como limitação permanente, não bug a corrigir depois.
- [ ] D-084-04 — Nenhuma ferramenta open source (patchright/Camoufox) dá garantia real contra Cloudflare Turnstile moderno; FlareSolverr (opção historicamente mais citada) está arquivado desde dez/2025. Decisão de manter os dois engines em cascata é experimental — dado real de sucesso por fonte decide o processo definitivo depois do primeiro smoke em beta.
- [ ] D-084-05 — RedeRPG (portal BR grande, `/downloads/` dedicada) exige cadastro/login — fora do MVP por trava de autenticação, não por falta de conteúdo relevante; candidato a spec futura se justificar o esforço de login automatizado (levanta questão própria de credenciais, fora do escopo atual).
- [ ] D-084-06 — RPG Grátis (`rpggratis.wordpress.com`) sem adapter funcional. Achado real confirmado via fetch direto de 2 posts durante a Fase 3: não existe formato consistente de "link de saída" no blog — um post tinha só embed de YouTube (zero link de material), outro tinha embed apontando pra Dungeonist Marketplace (site de venda, não confirma gratuidade nem é o arquivo em si). Sem seletor/padrão determinístico e seguro pra extrair automaticamente qual link é o material real — decisão do mantenedor foi pular por ora em vez de extrair o 1º link externo às cegas (risco de linkar destino errado/pago). Retomar só com estratégia dedicada (ex.: curadoria humana assistida, ou heurística mais sofisticada validada contra amostra maior de posts antes de automatizar).
- [ ] D-084-07 — Catarse (`catarse.me`) sem adapter funcional. Achado real confirmado via patchright real (não fetch simples — página é client-side rendered, precisa de headless mesmo pra descoberta, achado em si já diferente do assumido na spec original): `?category_id=4`/`?categoryId=4` não retornam projetos (filtro de categoria real do Catarse não identificado nesta sessão); link de projeto real segue padrão `/{slug}?ref=ctrse_..._project_card` na raiz do domínio (não `/pt/projects/{slug}` como assumido inicialmente). Sem tempo dedicado nesta sessão pra mapear a categoria/filtro correto de RPG — pendente de investigação futura antes de implementar o adapter.
- [ ] D-084-08 — Newton Rocha (`newtonrocha.wordpress.com`) sem adapter funcional. Mesmo achado estrutural do RPG Grátis (D-084-06): post testado ("fichas e novas versões dos sistemas de rpg... gratuitos para download", 2012) linka material via `goo.gl/6vreb2` (encurtador) — confirmado via HEAD real que o link AINDA resolve (302), mas aponta pra outra página do mesmo blog sem relação nenhuma com RPG (depoimentos de curso de inglês dos autores) — o link mudou de destino ou nunca foi o download real. Sem seletor/padrão confiável de "link de material" nesse blog (posts antigos usam encurtadores que podem apontar pra qualquer lugar anos depois) — mesma decisão do D-084-06: pular por ora, não extrair link às cegas.
